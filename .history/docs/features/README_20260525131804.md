# Features - Technical Implementation

This folder contains detailed technical documentation for each system capability.

## Purpose
- **MasterPlan/Features/** = Strategic view (problem, solution, why)
- **docs/features/** = Implementation view (how, code, specs)

## Structure

Each feature has:
```
feature-name/
├── README.md           - Overview and workflow
├── technology.md       - Tech stack, dependencies, rationale
├── metadata.md         - Data captured, schemas
├── optimization.md     - Performance tuning, targets
└── modules.md          - Code structure, API reference
```
