# me_workspace Phase Tracker

## Purpose

This is the working checklist for the project.

**Core Understanding: Platform First, Book Validates**

This project builds a **general-purpose document platform**, not just a book tool. The Health Fundamentals book is the **forcing function** that validates the platform works at scale, but the platform must serve multiple document types:

- **Books** - Long-form research (Health Fundamentals)
- **Business Plans** - Strategic documents
- **Strategic Plans** - Project roadmaps  
- **ThingsToDo** - Task and project management

**The Hierarchy:**
1. **Foundation** - Platform infrastructure (backups, security, database)
2. **Validation** - Health Fundamentals book (proves the platform works)
3. **Expansion** - Business plans, strategic plans (reuses proven platform)

**Critical Principle:** Platform stability is the prerequisite. If the foundation isn't solid, all document projects fail. Therefore, foundational infrastructure (immutable backups, database schema, safety guardrails) takes priority over book-specific features.

See [AcceptanceCriteria.md](AcceptanceCriteria.md) and [ArchitecturePrinciples.md](ArchitecturePrinciples.md) for the detailed philosophy.

## Phase 13: AI Safety Architecture

**Priority: 🔴 CRITICAL - Must exist before production use**

**Philosophy:** "Slow and steady" - reliability over speed, safety as architecture not afterthought

**Principle:** Safety must be deterministic, not probabilistic (baked into execution engine, not just prompts)

**The Railway Incident Lesson:** AI agents have deleted production databases despite explicit instructions. If backups use the same credentials as primary data, one delete command destroys everything. This phase exists to prevent that.

See [AcceptanceCriteria.md - Immutable Backup System](AcceptanceCriteria.md#foundation-immutable-backup-system-phase-13) for detailed acceptance criteria.
