## Structure

Each feature follows the **15 Decision Files** template:

| # | File | Purpose |
|---|------|---------|
| 00 | Feature-Index | Links to project/problem/solution |
| 01 | Problem-Definition | What pain point exists? |
| 02 | Solution-Approach | How do we solve it? |
| 03 | Competitive-Analysis | Market alternatives comparison |
| 04 | Target-Audience | End users, personas |
| 05 | Benefits | Value to company/users/system |
| 06 | Financial-Considerations | Cost, ROI, resources |
| 07 | Workflow | How it works (steps, diagrams) |
| 08 | Functionalities | What capabilities provided |
| 09 | Technology-Choices | Tech stack, rationale |
| 10 | UI-Design | Screen layout, UX |
| 11 | Attributes | Properties, metadata captured |
| 12 | Configuration-Parameters | Settings, tuning |
| 13 | Modules-Implementation | Code structure, APIs |
| 14 | Future-Vision | Roadmap, enhancements |
| 15 | Release-Timeline | Now vs later priorities |

**See [System Hierarchy](../System-Hierarchy.md)** for complete template specification.

## Universal Template

The **15 Decision Files** work for:
- ✅ **Features** (technical capabilities like chunking)
- ✅ **Problems** (user-facing challenges)
- ✅ **Projects** (books, business plans, reports)
- ✅ **Content Types** (journals, weekend projects, strategic plans)

**Principle:** Same structure, different scale. A weekend painting project has the same 15 files as a multi-year business plan, just with different depth.
