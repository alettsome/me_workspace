## Quick Summary

**Extends the existing numbered folder pattern** (`01-Inbox`, `02-Normalized`, etc.) with `00-Global-Inbox` for objective management.

Hierarchical objectives system with context-specific chat logs. Supports parent/child relationships, duration-agnostic tracking (3 minutes to 3 months), status updates, dependencies, milestones, and full conversation history per objective.

**Key Value**: Never lose context on what you're working on - each objective preserves its entire conversation history in its own folder.

## Builds On Existing System

**Current** (File Intake):
```
me_workspaces_runtime/
├── 01-Inbox/          ← Drop PDFs here, automatic processing ✅
├── 02-Normalized/     ← Processed files
├── 03-Chunked/        ← Chunked content
└── ...
```

**Adding** (Objective Management):
```
me_workspaces_runtime/
├── 00-Global-Inbox/   ← Drop objectives here, manual tracking 🔄
│   ├── obj-{id}/
│   │   ├── objective.json   ← Metadata
│   │   ├── notes.md         ← User notes
│   │   ├── chat/            ← Conversation logs
│   │   └── attachments/     ← Related files
├── 01-Inbox/          ← Existing file intake ✅
└── ...
```

**Same Philosophy**: Folder-as-agent, everything in one place, easy to navigate.

## Core Concepts

**Objectives (not tasks)**:
- Parent/child hierarchy (not taskmaster mentality)
- Duration-agnostic (short-term or long-term)
- Rich context (notes, status, dependencies, milestones)
- Conversation-backed (each has chat log)

**Context Preservation**:
- Each objective has corresponding chat/log folder
- All conversations about that objective stay with it
- Can revisit context months later

**Flexible Lifecycle**:
- Items can last 3 minutes or 3 years
- No forced completion dates
- Natural lifecycle (someday, active, complete, archived)

---

## Implementation Approach

**Hybrid**: Database + Filesystem (like Automatic-Processing)

- **Database**: Queryable metadata (status, dependencies, hierarchy)
- **Filesystem**: Rich content (notes, chat logs, attachments)
- **Dual-Write**: Keep both in sync
