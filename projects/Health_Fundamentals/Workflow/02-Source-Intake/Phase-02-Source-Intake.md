# Phase 2: Source Intake

## Goal
Create a repeatable way to bring different kinds of information into the system.

## Technology Used
- `Open WebUI`
- `ShareX`
- `PowerShell`
- `Windows Explorer`
- Manual copy and paste where simplest

## Source Types
- Chats
- PDFs
- Web articles
- Voice notes
- Video transcripts
- Business planning notes
- Lawfully reviewed books

## What Happens In This Phase
- A source enters the system
- The source gets labeled
- The source gets converted into text or notes
- The source becomes ready for summarization

## Required Metadata
- Source title
- Author or origin
- Date captured
- Rights label
- Source type
- Topic tags
- Page or section reference when relevant

## Checklist
- [ ] Create an intake folder for raw source notes
- [ ] Define a naming format for source files
- [ ] Define rights labels
- [ ] Export existing chats into text or markdown
- [ ] Gather public or owned PDFs
- [ ] Define when ShareX should be used
- [ ] Define when direct import is preferred over OCR
- [ ] Create a small first batch of 5 to 10 sources

## Ethical Rule
For rights-sensitive sources, use summary-only handling.

## Done Looks Like
- You have a labeled batch of traceable research inputs ready for processing
